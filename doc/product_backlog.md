# Product Backlog

## User Stories

| ID | Title | User Story | Priority | Size |
|---|---|---|---|---|
| US-01 | **Data Persistence** | As a manager, I want my inventory and menu data to be saved to a file so that I don't lose it when the program closes. | High | 8 |
| US-02 | **Modular Architecture** | As a developer, I want the codebase to be split into modules (Inventory, Menu, Cart) so that it is easier to maintain and extend. | High | 13 |
| US-03 | **Itemized Receipt** | As a customer, I want to see an itemized receipt with the names and prices of each item I purchased. | Medium | 5 |
| US-04 | **Automated Testing** | As a developer, I want to have automated tests for the core calculation logic to prevent regressions. | High | 8 |
| US-05 | **Input Validation** | As a manager, I want the system to validate user input (e.g., negative quantities) to prevent data corruption. | Medium | 3 |
| US-06 | **Dynamic Memory** | As a developer, I want the system to use dynamic memory allocation so that the inventory size is not limited to a fixed constant. | Low | 8 |

## Backlog Grooming Notes
- Prioritization is based on the immediate need for a stable and maintainable prototype.
- Sizing follows a Fibonacci-like scale (1, 2, 3, 5, 8, 13, 21).
